Logigramme du projet

Mise en place du moteur de recherche textuel

Une lecture en colonne du pipeline : chaque bloc correspond à une étape réelle du projet, avec son intention à gauche et ce qui a été fait à droite.

Mise à jour : 16 juin 2026 Format : logigramme Projet : moteur de recherche de brevets
Entrée Un CSV de brevets contenant notamment les titres et abstracts.
Traitement Nettoyage, index inversé, scoring TF-IDF, BM25 et variantes WordNet.
Évaluation Comparaison top 10, temps moyen, familles de requêtes et requêtes à surveiller.
Sortie Une interface Streamlit pour chercher, comparer et générer une lecture rapide.
01

Choisir le corpus

Objectif

Partir d'une base documentaire assez riche pour tester une recherche textuelle.

Ce qui est fait

Le projet utilise un dataset Kaggle de brevets de santé. Les colonnes utiles sont surtout Title et Abstract.

02

Nettoyer les données

Objectif

Éviter d'indexer des textes vides, absents ou inutilisables.

Ce qui est fait

cleaner.py lit le CSV par lots, retire les abstracts manquants et conserve les textes détectés comme anglais avec langdetect.

03

Créer les documents

Objectif

Transformer chaque brevet en élément simple à indexer et à retrouver.

Ce qui est fait

Dans main.py, chaque brevet reçoit un doc_id. Le texte indexé est construit par build_search_text avec les colonnes Title et Abstract.

04

Normaliser le texte

Objectif

Faire en sorte que les mêmes mots soient reconnus malgré la casse ou la ponctuation.

Ce qui est fait

_nettoyer_texte passe le texte en minuscules, remplace la ponctuation par des espaces, puis découpe le texte en mots.

05

Construire l'index inversé

Objectif

Retrouver rapidement les documents contenant un mot sans parcourir tout le corpus.

Ce qui est fait

indexer_corpus stocke pour chaque terme une liste de couples (doc_id, fréquence). Le moteur garde aussi la taille de chaque document et calcule la longueur moyenne du corpus.

06

Filtrer les candidats

Objectif

Garder uniquement les documents qui correspondent à tous les mots de la requête.

Ce qui est fait

requete_and intersecte les listes de documents des mots demandés avec un algorithme à deux pointeurs.

07

Classer avec TF-IDF

Objectif

Mettre en premier les documents où les mots de la requête sont importants.

Ce qui est fait

chercher_tfidf combine la fréquence du mot dans le document avec sa rareté dans le corpus, puis trie les résultats par score décroissant.

08

Classer avec BM25

Objectif

Obtenir un classement plus robuste pour des documents de longueurs différentes.

Ce qui est fait

chercher_bm25 ajoute une saturation de fréquence et une normalisation par longueur. Les paramètres utilisés sont k1 = 1.5 et b = 0.75.

09

Lemmatiser

Objectif

Rapprocher des formes comme robot, robots, clean et cleaning.

Ce qui est fait

Le moteur construit un index lemmatisé avec WordNet. Si la ressource manque, il revient automatiquement à l'index classique.

10

Ajouter les synonymes

Objectif

Retrouver un brevet même si la requête emploie un vocabulaire proche mais différent.

Ce qui est fait

Les synonymes WordNet sont ajoutés côté requête. Chaque mot devient un petit groupe de termes équivalents, puis le moteur score le meilleur terme du groupe.

11

Créer l'interface

Objectif

Permettre d'utiliser le moteur sans manipuler directement le code Python.

Ce qui est fait

main.py fournit une application Streamlit avec des onglets pour TF-IDF, BM25, synonymes, lemmatisation et résultats détaillés.

12

Comparer les méthodes

Objectif

Mesurer quelle méthode retrouve le mieux un brevet cible selon plusieurs formulations.

Ce qui est fait

Le dashboard teste des variantes de requête, calcule le rang du brevet cible, le taux de réussite, la présence dans le top choisi et une synthèse fixe en top 10.

13

Mesurer les performances

Objectif

Comparer les méthodes avec des indicateurs lisibles, pas seulement avec des scores internes.

Ce qui est fait

build_comparison_dataframe mesure le temps de chaque recherche avec time.perf_counter, puis le résumé affiche taux de réussite, taux top 10, rang moyen et temps moyen.

14

Classer les requêtes

Objectif

Comprendre pourquoi une méthode réussit ou échoue selon le type de formulation.

Ce qui est fait

classify_query_family range les requêtes en familles : exacte, ordre des mots, pluriel, reformulation sémantique, requête large ou requête bruitée.

15

Produire une conclusion

Objectif

Aider à choisir une méthode en donnant une lecture métier rapide du dashboard.

Ce qui est fait

Le dashboard classe les méthodes par réussite, top 10, rang moyen et temps moyen. Il ajoute une lecture rapide, une pénalité de bruit et une conclusion automatique.

Commandes utiles

Installer les dépendances, télécharger WordNet, puis lancer l'application Streamlit.

pip install -r requirements.txt python -m nltk.downloader wordnet python -m nltk.downloader omw-1.4 python3 -m streamlit run main.py
L'application attend le fichier data/patent_analysis_data.csv. Si le fichier n'est pas présent, il faut placer le CSV nettoyé à cet emplacement.

Métriques du dashboard

  • Taux de réussite : part des requêtes où le brevet cible est retrouvé.
  • Taux top 10 : part des requêtes où le brevet cible apparaît dans les 10 premiers résultats.
  • Rang moyen : position moyenne du brevet quand il est retrouvé ; plus c'est bas, mieux c'est.
  • Temps moyen : durée moyenne d'une recherche pour une méthode.
  • Pénalité bruit : signal d'alerte quand les synonymes ou requêtes larges peuvent favoriser des résultats moins spécifiques.

Fichiers importants

  • src/lib/MoteurRechercheTextuel.py : logique du moteur de recherche.
  • src/lib/cleaner.py : nettoyage et filtrage du CSV.
  • main.py : interface Streamlit et dashboard de comparaison.
  • src/exemple/ : notebooks d'exemples et d'expérimentation.
